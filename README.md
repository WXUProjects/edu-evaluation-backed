# 无锡学院-新西伯利亚学院评教系统
## 需求分析
目的：做一套学生评教系统，包含：管理端、学生评教端。  
由于未来可能用户数将会是1000名左右，考虑极端情况下，要能够抗住1000次/s的高并发。
### 学生评教端（以小程序方式呈现）
- 学生登录功能
- 能够获取学生对应的课程
- 能够获取课程对应的教师
- 对相应的教师进行评教

### 管理端
- 登录功能
- 创建班级，导入学生名单
- 创建课程，导入教师名单（允许包含一些教师的介绍）
- 按照班级，选择对应的课程。
- 查看、导出相应的评教结果。

## 技术栈选型
### 后端
- Golang + pgsql + go-kratos + redis

### 前端
- Uniapp + ts + vue
